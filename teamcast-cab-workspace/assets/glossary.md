# Glossary

> Shared terminology for the Teamcast CAB engine.

---

| Term | Definition |
|------|-----------|
| **CAB** | Customer Advisory Board — a group of leaders invited to provide product feedback and strategic input. Used here as a soft-entry sales motion. |
| **ICP** | Ideal Customer Profile — the definition of who we want to reach (company + contact characteristics). |
| **TAM** | Total Addressable Market — the total pool of potential prospects matching our filters. |
| **Personalized Line** | A specific, source-backed observation about a prospect inserted into the email template. |
| **Signal** | A verifiable fact about a prospect or their company (hiring, funding, expansion, growth) used for personalization. |
| **Confidence Score** | A 1-10 rating of how reliable a data point or signal is. |
| **Suppression List** | A list of emails and domains that must not be contacted. |
| **QA Pass Rate** | Percentage of drafted emails that pass quality assurance review. |
| **Bounce Rate** | Percentage of sent emails that failed to deliver. |
| **Positive Reply Rate** | Percentage of sent emails that received a positive (interested) reply. |
| **Batch** | A set of prospects processed together in one campaign run. |
| **Sequence** | The ordered set of emails a contact receives (Email 1/3 → Email 2 follow-up). |
| **Fallback to Generic** | When no credible personalization signal is found, using an honest generic framing instead. |
| **Seniority Score** | A numerical ranking (1-10) of a contact's organizational seniority. |
| **Campaign Fit Reason** | A brief explanation of why a prospect matches the ICP. |
| **Enrichment** | The process of finding additional data about a prospect (email, company info, growth data). |
| **Deduplication** | Removing duplicate contacts to prevent double-emailing. |
| **Mailbox Rotation** | Distributing sends across multiple email accounts to protect domain reputation. |
| **Warmup** | Gradually increasing send volume on a new mailbox to build sender reputation. |
| **Deliverability** | The likelihood that an email reaches the recipient's inbox (not spam). |
| **Reply Taxonomy** | The classification system for categorizing inbound replies. |
| **Escalation** | Routing an issue to a human operator for decision. |
| **Heartbeat** | The recurring daily/weekly checklist that drives campaign operations. |
| **Workspace** | An OpenClaw operating environment with its own files, agents, memory, and tools. |
| **Sub-agent** | A specialized agent within the workspace that handles a specific part of the pipeline. |
| **Skill** | A reusable procedure in OpenClaw that an agent can invoke. |
